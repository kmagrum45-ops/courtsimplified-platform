# PROJECT DOCTRINE

CourtSimplified is not:

- a chatbot
- a form filler
- a document generator
- a legal search tool
- a lawyer CRM

CourtSimplified is a Litigation Operating System.

Its purpose is to become the unified operating environment for self-represented litigants.

The system must eventually manage:

- litigation
- procedure
- evidence
- proof
- legal authority
- issue detection
- contradiction detection
- credibility analysis
- procedural intelligence
- document generation
- court package generation
- settlement preparation
- conference preparation
- motion preparation
- trial preparation
- authority verification
- workflow orchestration

Every system must support that objective.

No system may be created that operates independently from the Litigation Operating System architecture.

---

# MASTER RULE

CourtSimplified must operate as a single unified intelligence platform.

No duplicate engines.

No parallel workflows.

No competing sources of truth.

No architecture drift.

All intelligence must ultimately flow through the Master Case architecture.
